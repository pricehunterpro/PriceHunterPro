✅ Funcionalidades Implementadas

🖥️

Dashboard Web Angular 17

Listado de ofertas con filtros por tienda, categoría, marca y descuento mínimo. Búsqueda por palabra clave con regex de límite de palabra (evita matches como "puma" en "espuma"). Ordenamiento por descuento, margen, precio y diferencia de mercado.

🔔

Alertas de precio por debajo del mercado

Detecta productos cuyo precio actual es al menos 15% menor a su promedio histórico propio (mínimo 2 registros con +12h de antigüedad). Envía alerta al canal Telegram automáticamente post-scraping con deduplicación Redis (TTL 24h).

📢

Bot Telegram — Canal DEV y Canal PRD

Publica simultáneamente en canal familiar (DEV) y canal público (PRD). Formato con foto del producto, precio original tachado, precio actual, % descuento y link. Publicación automática 3 veces al día: 8:00am · 12:05pm · 8:05pm.

🎬

Generador de Imágenes TikTok (Pillow)

Genera imágenes 1080×1920 (formato vertical TikTok) con: barra de tienda en color corporativo, foto del producto descargada del URL, badge de descuento en rojo, precio original tachado, precio actual en verde, ahorro en píldora verde, CTA y hashtags. Se envía al Telegram del admin a las 7:00am con caption listo para pegar en TikTok.

🗄️

Base de datos con historial de precios

Modelos: Product, StoreProduct, PriceHistory, ScrapingLog. Detección de "below market" con ventana temporal de 12h. Limpieza automática de historial mayor a 30 días. Deduplicación por store_sku.

🌐

Acceso público vía ngrok

Túnel ngrok activo en https://energetic-respect-sixth.ngrok-free.dev. CORS configurado para permitir origen wildcard compatible con ngrok. nginx con Docker DNS resolver para evitar cache de IPs de contenedores.

⚡

API REST FastAPI + Endpoints manuales

POST /api/v1/deals/scrape/trigger — scraping manual
POST /api/v1/deals/telegram/publish — publicar deals en Telegram
POST /api/v1/deals/tiktok/generate — generar imagen TikTok
GET /api/v1/deals — listar deals con filtros
GET /api/v1/deals/stats — estadísticas globales

🐛 Problemas Resueltos

Dockerfile faltante

El backend no tenía Dockerfile. Creado con Python 3.11-slim, dependencias de sistema para Playwright, instalación de Chromium headless.

nginx "Connection refused" post-restart

nginx cacheaba la IP del contenedor backend. Solución: resolver 127.0.0.11 + variable set $backend para forzar re-resolución DNS.

Telegram "token o canal vacío"

El contenedor tenía nombres de variables obsoletos. Solución: docker compose up --force-recreate (no restart) para releer el .env.

Búsqueda "puma" devolvía "espuma"

El match era substring. Solución: regex con word-boundary r'\b' + re.escape(q) para buscar solo palabras completas.

Canal PRD no publicaba

El bot no era admin del canal. Pasos: añadir bot como admin en Telegram → obtener Channel ID con getUpdates → actualizar .env y recrear contenedores.

Ofertas especiales no aparecían

Los scrapers solo buscaban por categoría. Añadida función _scrape_top_deals() que obtiene los 300 mejores descuentos sin filtro de categoría.
